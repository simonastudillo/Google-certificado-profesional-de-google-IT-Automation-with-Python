## Curso 2 Glosario
- `Absolute path` (Ruta absoluta): Ruta completa al recurso en el sistema de archivos.
- `Alteration` (Alteración): Expresión regular que coincide con cualquiera de las alternativas separadas por el símbolo de barra vertical (|).
- `Automated testing` (Pruebas automáticas): Proceso mediante el cual el software se autoevalúa en busca de errores y confirma su correcto funcionamiento.
- `Backreference` (Retroreferencia): Se aplica al usar `re.sub()` para sustituir el valor de un grupo de captura en la salida.
- `Bash` (Bash): El intérprete de comandos más utilizado en Linux.
- `Bash script` (Script de Bash): Un script que contiene varios comandos.
- `Black-box testing` (Pruebas de caja negra): Pruebas en las que se sabe qué se supone que debe hacer el programa, pero no cómo lo hace.
- `Character classes` (Clases de caracteres): Se escriben entre corchetes y nos permiten listar los caracteres que queremos que coincidan dentro de ellos.
- `Character ranges` (Rangos de caracteres): Rangos que se utilizan para comparar un solo carácter con un conjunto de posibilidades.
- `Comma-separated values (CSV)` (Valores separados por comas): Un formato de datos muy común que se utiliza para almacenar datos como segmentos de texto separados por comas.
- `Command-line arguments` (Argumentos de línea de comandos): Entradas que se proporcionan a un programa al ejecutarlo desde la línea de comandos.
- `Compiled language` (Lenguaje compilado): Código compatible que se puede ejecutar en diferentes plataformas.
- `Cut` (Cortar): Un comando que divide y toma solo partes de cada línea usando espacios.
- `Dialects` (Dialectos): Reglas que definen cómo se estructura un archivo CSV.
- `Edge cases` (Casos límite): Entradas al código que producen resultados inesperados, ubicadas en los extremos de los rangos de entrada.
- `Environment variables` (Variables de entorno): Configuraciones y datos almacenados fuera de un programa al que este puede acceder para modificar su comportamiento en un entorno específico.
- `File systems` (Sistemas de archivos): Métodos y estructuras utilizados para organizar y controlar cómo se almacenan y acceden los datos.
- `Globs` (Globs): Caracteres que crean listas de archivos, como el asterisco y el signo de interrogación.
- `grep` (grep): Una herramienta especialmente fácil de usar pero extremadamente potente para aplicar expresiones regulares (RegExes).
- `Input/Output (I/O)` (Entrada/Salida (E/S)): Estos flujos son el mecanismo básico para realizar operaciones de entrada y salida en los programas.
- `Integrated Development Environment (IDE)` (Entorno de Desarrollo Integrado (IDE)): Un editor de código con capacidades adicionales para simplificar la escritura de scripts.
- `Interpreter` (Intérprete o lenguaje interpretado): Un programa intermedio utilizado para ejecutar las instrucciones especificadas en el código.
- `Kernel` (Núcleo): El núcleo principal de un sistema operativo que se comunica directamente con el hardware y gestiona los recursos del sistema.
- `Linux` (Linux): Un sistema operativo de código abierto cuyo software se puede compartir, modificar y distribuir libremente.
- `Log files` (Archivos de registro): Los archivos de registro son registros o archivos de texto que almacenan un historial de eventos, acciones o errores generados por un sistema informático, software o aplicación con fines de diagnóstico, resolución de problemas o auditoría.
- `Lookahead` (Adelanto): Expresión regular que coincide con un patrón solo si le sigue otro patrón.
- `Mac OS` (Mac OS): Sistema operativo desarrollado por Apple: El formato que controla lo que se puede hacer con un archivo recién abierto.
- `Operating System (OS)` (Sistema operativo (SO)): Software que gestiona todo lo que sucede en la computadora, compuesto por dos partes principales: el núcleo y el espacio de usuario.
- `Pipes` (Pipas): Proceso para conectar la salida de un programa con la entrada de otro.
- `Piping` (Piping): Proceso para conectar varios scripts, comandos u otros programas en una canalización de procesamiento de datos.
- `Pytest` (Pytest): Una potente herramienta de prueba de Python que ayuda a los programadores a escribir programas más eficaces y estables.
- `Qwiklabs` (Qwiklabs): Un entorno de aprendizaje en línea o máquina virtual para simular escenarios del mundo real.
- `Reader objects` (Objetos lectores): Objeto que representa un elemento o entidad dentro de una escena que necesita ser renderizado en la pantalla.
- `Redirection` (Redirección): Proceso para enviar un flujo de datos a un destino diferente.
- `Regular expression` (Expresión regular): Una consulta de búsqueda de texto expresada mediante un patrón de cadena, también conocida como RegEx o RegExp.
- `Relative path` (Ruta relativa): Parte de una ruta que muestra la ubicación del recurso en relación con el directorio de trabajo actual.
- `Signals` (Señales): Tokens que se envían a los procesos en ejecución para indicar una acción deseada.
- `Software testing` (Pruebas de software): Proceso de evaluación de código informático para determinar si funciona como se espera.
- `Standard input (STDIN)` (Entrada estándar (STDIN)): Canal de comunicación entre un programa y una fuente de entrada.
- `Standard output (STDOUT)` (Salida estándar (STDOUT)): Vía de comunicación entre un programa y un destino de salida, como una pantalla.
- `Standard error (STDERR)` (Error estándar (STDERR)): Muestra la salida de forma similar a la salida estándar, pero se utiliza específicamente como canal para mostrar mensajes de error y diagnósticos del programa.
- `Shell` (Shell): Aplicación que lee y ejecuta todos los comandos.
- `Subprocesses` (Subprocesos): Proceso para llamar y ejecutar otras aplicaciones desde Python, incluyendo otros scripts de Python.
- `Test case` (Caso de prueba): Unidad de prueba individual que busca una respuesta específica a un conjunto de entradas.
- `Test environment` (Entorno de prueba): Preparado para realizar una o más pruebas.
- `Test suite` (Suite de pruebas): Conjunto de pruebas que se ejecutan conjuntamente.
- `Test runner` (Ejecutor de pruebas): Ejecuta la prueba y proporciona a los desarrolladores los datos del resultado.
- `unittest` (unittest): Conjunto de herramientas de Python para construir y ejecutar pruebas unitarias.
- `Unit tests` (Pruebas unitarias): Pruebas para verificar que pequeñas partes aisladas de un programa funcionan correctamente.
- `User space` (Espacio de usuario): Todo lo que está fuera del núcleo con lo que los usuarios interactúan directamente.
- `Virtual environment` (Entorno virtual): Una herramienta que permite crear entornos aislados para proyectos de Python.
- `Virtual machine (VM)` (Máquina virtual (VM)): Un ordenador simulado mediante software.
- `White-box testing` (Prueba de caja blanca): Una prueba en la que el creador de la prueba conoce el funcionamiento del código y puede escribir casos de prueba que, basándose en ese conocimiento, garantizan su correcto funcionamiento.
- `Wildcard` (Comodín): Un carácter que puede coincidir con más de un carácter.
- `Windows operating system` (Sistema operativo Windows): Sistema operativo desarrollado por Microsoft.
- `Writer objects` (Objetos de escritura): La capacidad de escribir datos en un archivo CSV.
