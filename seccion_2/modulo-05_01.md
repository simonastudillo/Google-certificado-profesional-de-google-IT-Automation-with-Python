# Test simples

## Introducción al módulo 5: Testing en Python 
- En este módulo, nos adentraremos en cómo podemos crear ​pruebas automáticas que ​nos llevarán a cabo este tipo de comprobaciones. 
- Esto nos permite ​concentrarnos en escribir el código en lugar de ​verificar si algún cambio que hagamos en ​él rompe la funcionalidad anterior
- También nos ayudará a verificar que las características que ​agregamos hagan lo que esperamos de muchas maneras posibles
- Aprenderemos sobre los diferentes tipos de ​pruebas que existen, ​y cómo podemos usarlas para hacer que nuestro código sea más confiable ​y, finalmente, también aprenderemos cómo ​manejar errores y excepciones en el código Python.

---

## Qué es testing?
- Las pruebas de software son un proceso de evaluación de ​código informático para determinar ​si hace o no lo que espera que haga
- Al probar una pieza de software, ​desea encontrar los errores y ​defectos y ver dónde van mal las cosas
- Los scripts y los programas pueden ​fallar de todo tipo de maneras extrañas, ​especialmente a medida que se vuelven más complicados
- Las pruebas pueden ayudar a que el buen código sea genial
- 3 tipos de testing:
  - Pruebas unitarias: se centran en probar una unidad de código a la vez, como una función o un método.
  - Pruebas de integración: se centran en probar cómo interactúan diferentes unidades de código entre sí.
  - Pruebas funcionales: se centran en probar el comportamiento del software desde la perspectiva del usuario final.
